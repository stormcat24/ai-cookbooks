変更内容を分析し、適切なコミットメッセージを提案してコミットするスキル。PRは作成せず、コミットのみを行う。

## 手順

1. `git status` と `git diff`（ステージ済み・未ステージ両方）で現在の変更内容を確認する
   - 変更がない場合はその旨を伝えて終了する
2. `git log --oneline -5` で直近のコミットメッセージのスタイルを確認する
3. 変更内容を分析し、適切なコミットメッセージを提案する
   - `$ARGUMENTS` が指定されていればそれをメッセージの参考にする
   - コミットメッセージは変更の「なぜ（why）」に焦点を当てる
   - リポジトリの既存のコミットメッセージスタイルに合わせる
4. ユーザーにコミットメッセージを確認する
5. 変更をステージングしてコミットする
   - `.env`、認証情報、秘密鍵などの機密ファイルは除外する。該当ファイルがあればユーザーに報告する
   - `git add -A` ではなく、対象ファイルを明示的に指定する
   - ユーザーが特定のファイルだけコミットしたい場合はその指示に従う
6. コミット完了を報告する（コミットハッシュと変更の概要を表示する）
