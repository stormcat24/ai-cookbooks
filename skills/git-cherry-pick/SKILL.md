特定のコミットを現在のブランチにcherry-pickするスキル。コンフリクト時の対処も案内する。

## 手順

1. cherry-pick対象のコミットを特定する
   - `$ARGUMENTS` にコミットハッシュやブランチ名が指定されていればそれを使用する
   - 指定がなければ `git log --oneline` でコミット一覧を表示し、ユーザーに選択させる
   - ブランチ名が指定された場合は `git log --oneline <branch>` でそのブランチのコミットを表示する
2. `git show --stat <commit-hash>` で対象コミットの内容を表示し、ユーザーに確認する
3. 現在のブランチと作業状態を確認する
   - 未コミットの変更がある場合は先にコミットまたはstashするよう案内する
4. `git cherry-pick <commit-hash>` を実行する
5. 結果に応じて対応する
   - **成功した場合**: 完了を報告する（取り込まれたコミットのハッシュと内容を表示）
   - **コンフリクトが発生した場合**:
     - コンフリクトしているファイルを表示する
     - ユーザーに選択肢を提示する: コンフリクトを解決して続行する / `git cherry-pick --abort` で中止する
     - 解決を選んだ場合はコンフリクト箇所を表示し、解決をサポートする
     - 解決後に `git add` → `git cherry-pick --continue` で完了する
6. 複数コミットをcherry-pickする場合は、1つずつ順番に実行する（一括で `git cherry-pick A B C` はコンフリクト時の対処が複雑になるため避ける）
