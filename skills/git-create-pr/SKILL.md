mainブランチ上の変更や未整理の作業を、ブランチ作成→コミット→PR作成まで一気通貫で行うスキル。各ステップでユーザーの確認を取りながら進めることで、意図しないPRが作られるのを防ぐ。

## 手順

1. `git status` と `git diff` で現在の変更内容を確認する
   - 変更がない場合はその旨を伝えて終了する
   - すでにmain以外のブランチにいる場合は、新しいブランチを作るか現在のブランチでPRを作るかユーザーに確認する
2. 変更内容を分析し、適切なブランチ名を決定する
   - `$ARGUMENTS` が指定されていればそれをブランチ名に使用
   - ブランチ名は `feature/xxx`, `fix/xxx`, `chore/xxx` のようにプレフィックスを付ける
   - ユーザーへの確認は不要。変更内容から判断して自律的に決定する
3. 新しいブランチを作成してチェックアウトする
4. 変更内容を分析し、適切なコミットメッセージを作成する
   - コミットメッセージは変更の「なぜ（why）」に焦点を当てる。diffを見ればwhatはわかるので、動機や背景を伝える方がレビュアーにとって価値がある
5. ユーザーにコミットメッセージを確認する
6. 変更をステージングしてコミットする
   - `.env`、認証情報、秘密鍵などの機密ファイルは除外する。該当ファイルがあればユーザーに報告する
   - `git add -A` ではなく、対象ファイルを明示的に指定する
7. リモートにプッシュする（`git push -u origin <branch-name>`）
8. `gh pr create` でPull Requestを作成する
   - PRタイトルは70文字以内で簡潔に
   - PR本文のフォーマット:
     ```
     ## Summary
     - 変更の要約（1-3行）

     ## Test plan
     - テスト手順のチェックリスト
     ```
9. 作成されたPRのURLをユーザーに報告する
