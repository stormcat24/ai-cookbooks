Pull Requestの差分を取得し、コードレビューを実施するスキル。バグ、セキュリティリスク、改善点を指摘する。

## 手順

1. レビュー対象のPRを特定する
   - `$ARGUMENTS` にPR番号やURLがあればそれを使用する
   - 指定がなければ `gh pr list` で一覧を表示し、ユーザーに選択させる
2. `gh pr view <PR番号> --json title,body,author,baseRefName,headRefName,files` でPRの概要を取得する
3. `gh pr diff <PR番号>` で差分を取得する
4. 以下の観点でレビューを行う
   - **バグ・ロジックエラー**: 境界条件、null/undefined、型の不一致、off-by-oneなど
   - **セキュリティ**: インジェクション、機密情報のハードコード、認証・認可の抜けなど
   - **設計・可読性**: 命名、責務の分離、不要な複雑さ、重複コードなど
   - **テスト**: テストの有無、カバレッジの十分さ、エッジケースの考慮
   - **良い点**: 良い設計判断やきれいな実装があれば積極的に言及する
5. レビュー結果を以下のフォーマットで報告する
   - 全体の所感（1-2文）
   - 指摘事項をカテゴリ別にリスト化（重要度: Critical / Warning / Suggestion）
   - 良い点があれば記載
6. ユーザーが希望すれば `gh pr review <PR番号>` でGitHub上にレビューコメントを投稿する
   - 投稿前に必ず内容をユーザーに確認する
