作業ブランチからmainに安全に戻り、リモートの最新状態に同期するスキル。未コミットの変更がある場合は失わないように確認してから切り替える。

## 手順

1. `git status` で現在の状態を確認する
   - リベースやマージが進行中の場合は、その旨をユーザーに伝えて指示を仰ぐ（勝手にabortしない）
2. 未コミットの変更がある場合はユーザーに警告し、どうするか選択させる
   - stashする / コミットする / 変更を破棄する
   - 変更の破棄は取り消せないため、ユーザーが明示的に選んだ場合のみ実行する
3. mainブランチにチェックアウトする
4. `git pull` でリモートの最新を取得する
5. 同期完了を報告する（現在のHEADコミットを表示すると、ユーザーが最新状態を確認しやすい）
