マージ済みのローカルブランチを安全に削除してリポジトリを整理するスキル。削除前にユーザーの確認を必ず取る。

## 手順

1. `git fetch --prune` でリモートの最新情報を取得し、削除済みリモートブランチの追跡参照を消す
2. `git branch --merged main` でmainにマージ済みのローカルブランチを一覧表示する
   - `main` と現在チェックアウト中のブランチは除外する
3. マージ済みブランチがない場合はその旨を伝えて終了する
4. 削除対象のブランチ一覧をユーザーに表示し、確認を取る
   - 各ブランチの最後のコミット日時も表示すると判断しやすい
   - ユーザーが一部のブランチを残したい場合はその指示に従う
5. 確認が取れたブランチを `git branch -d` で削除する
   - `-D`（強制削除）は使わない。マージされていないブランチが混ざっていた場合にエラーで検知できる
6. 削除結果を報告する（削除したブランチ数と名前）
7. リモートにも不要なブランチがあれば、その旨をユーザーに伝える（自動では削除しない）
